Setting Up Octopress

Jun 22nd, 2012

I spent the last past two days working on setting up my website. My first effort involved using Jekyll and the second involved using Octopress, a package which is built on top of Jeykll and provides some extra plugins, a new deployment system, and a built-in template.

My main goal was to take advantage of the cool auto-generating features of Github Pages, which provides free hosting and an automated system which can turn a collection of text files created in writer-friendly text formats like Textile or Markdown into a static HTML website. The site is regenerated after a git push.

It is nice in practice, but I haven’t hand-written HTML and CSS in a while, and getting all of the content to display the way that I wanted it took some effort. In Jekyll, the layout code more or less has to be done from stratch. In Octopress, there is an attractive default theme with a number of nice features, but modifying it to avoid looking exactly like everyone else’s Octopress blog takes some time. Rather than using CSS, it uses a CSS-generating system called SASS to generate the layouts. If you have some understanding of CSS, it is not so hard to make pick up; I have been outside of the web developer world for a while and found there were a number of things I needed to learn at once.

All said and done, my goal of putting up a quick website turned out to be not particularly quick, but this is the kind of system that a web designer might like, and that gives me a chance to see what has been going on in that field recently.
